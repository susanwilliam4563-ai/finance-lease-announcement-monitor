# 融资租赁公告监测台

用于持续收集、检索和复核上市公司与发债主体融资租赁相关公告。历史数据从 `2021-01-01` 开始。

## 本机使用

```bash
python3 -m pip install -r requirements.txt
python3 server.py
```

浏览器打开：

```text
http://127.0.0.1:8765/
```

不要直接双击 `index.html` 作为日常入口。直接打开文件只会显示静态快照，无法自动获取新公告。

## 自动更新

网站包含两层自动更新：

- 后台默认每 60 分钟核对最近 14 天的公告，写入 SQLite 数据库，并更新静态快照。
- 每位访客的页面默认每 60 秒检查一次数据版本；发现新记录后自动重新载入，无需点击按钮。

新增公告会尝试读取 PDF 正文，提取金额、期限、交易对手和证据片段。历史快速回填产生的“待补充正文”记录，会在后续日常刷新覆盖范围内逐步补齐。

可通过环境变量调整：

```bash
AUTO_REFRESH_INTERVAL_MINUTES=30 PAGE_POLL_INTERVAL_SECONDS=60 python3 server.py
```

关闭后台自动采集：

```bash
AUTO_REFRESH_ENABLED=0 python3 server.py
```

## 数据来源现状

当前已自动接入的主数据源是巨潮资讯公开公告。交易所公告通过巨潮披露索引覆盖 A 股市场；公司官网、预警通、发债披露、互联网和公众号目前保留复核入口与统一数据结构，但尚未实现无人值守采集。

界面会明确显示实际接入的来源数量，不把外部检索入口计入已采集数据。

## 给其他人访问

`127.0.0.1` 只允许本机访问，不能直接发给其他人。本项目支持两种公网方式。

### 无服务器方式：GitHub Pages

适合没有云服务器、接受网页代码与市场公开公告数据公开存放的场景：

- GitHub Pages 提供固定公网网址。
- GitHub Actions 每小时第 17 分钟执行增量抓取并重新发布。
- 网页每 60 秒检查一次 `data/status.json`，发现新版本后自动加载 `data/records.json`。
- 云端只发布网页和公开公告 JSON，不发布 SQLite 数据库、后台程序或本机文件。
- `.github/workflows/update-pages.yml` 是自动更新与发布流程。

GitHub 定时任务不是精确实时任务，高峰期可能延迟。第一次发布还需要在仓库设置中允许 GitHub Pages 使用 Actions 发布。

### 服务器方式：容器部署

适合需要私有仓库、更高刷新稳定性或后续接入登录权限的场景。

项目已包含容器部署文件：

```bash
docker compose up -d --build
```

公网部署默认行为：

- 后台每 60 分钟自动采集。
- 所有访问者页面每 60 秒自动同步。
- 访客只能读取和搜索，公网环境关闭手动刷新接口。
- SQLite 数据库保存在服务器的 `data` 目录，服务重启后不会丢失。
- 服务器需保持运行并允许访问巨潮资讯和东方财富公开接口。

正式对外使用时，可再配置自定义域名。无论选择哪种方式，发布动作都需要对应托管平台的账号与权限。

## 维护命令

首次历史回填：

```bash
python3 cninfo_updater.py --start 2021-01-01 --skip-pdf
```

补齐注册地和证监会行业：

```bash
python3 eastmoney_profile_enricher.py
```

导出静态快照：

```bash
python3 export_records_js.py
```

抽取规则升级后重跑指定时间段：

```bash
python3 cninfo_updater.py --start 2026-07-01 --refresh-existing
```

## 主要能力

- 按公司、标题、交易对手、金额、证据片段和关键词检索。
- 按时间、地区、证监会行业、主体类型、来源、风险和复核状态筛选。
- 事件列表与右侧证据链联动，并支持事件专属链接。
- 展示公告原文、PDF、命中片段、业务要素完整度和后续复核建议。
- 导出当前筛选结果为 CSV。
- 显示数据核对截止日、后台任务状态、下次采集时间和来源覆盖情况。
