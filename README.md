# 融资租赁公告监测台

用于持续收集、检索和复核上市公司与发债主体融资租赁相关公告。历史数据从 `2021-01-01` 开始。

## 本机使用

```bash
python3 -m pip install -r requirements.txt
python3 server.py
```

浏览器打开：

```text
http://127.0.0.1:8765/
```

不要直接双击 `index.html` 作为日常入口。直接打开文件只会显示静态快照，无法自动获取新公告。

## 自动更新

网站包含两层自动更新：

- 后台默认每 60 分钟核对最近 14 天的公告，写入 SQLite 数据库，并更新静态快照。
- 每位访客的页面默认每 60 秒检查一次数据版本；发现新记录后自动重新载入，无需点击按钮。

新增公告会尝试读取 PDF 正文，提取金额、期限、交易对手和证据片段。历史快速回填产生的“待补充正文”记录，会在后续日常刷新覆盖范围内逐步补齐。

可通过环境变量调整：

```bash
AUTO_REFRESH_INTERVAL_MINUTES=30 PAGE_POLL_INTERVAL_SECONDS=60 python3 server.py
```

关闭后台自动采集：

```bash
AUTO_REFRESH_ENABLED=0 python3 server.py
```

## 数据来源现状

当前直接自动采集巨潮资讯和港交所披露易。上交所、深交所和北交所公告通过巨潮披露索引覆盖，页面会标记为“巨潮覆盖”，不会伪装成独立直连。公司官网由公开公司画像提供复核入口，不作为已经自动采集的公告来源。互联网、公众号、预警通和发债披露在没有稳定公开 API 或授权数据接口前标记为“未自动接入”。

主体画像使用公开公司资料补充注册地、证监会行业、官网和实际控制人；企业性质根据公司简介中的明确表述及实际控制人规则判定，同时保留判定依据。总市值优先读取东方财富公开行情，失败时使用腾讯证券公开行情，并单独记录市值日期和来源。

界面会明确显示实际接入的来源数量，不把外部检索入口计入已采集数据。

## 给其他人访问

`127.0.0.1` 只允许本机访问，不能直接发给其他人。本项目支持两种公网方式。

### 无服务器方式：GitHub Pages

适合没有云服务器、接受网页代码与市场公开公告数据公开存放的场景：

- GitHub Pages 提供固定公网网址。
- GitHub Actions 每小时第 17 分钟执行增量抓取并重新发布。
- 网页每 60 秒检查一次 `data/status.json`，发现新版本后自动加载 `data/records.json`。
- 云端只发布网页和公开公告 JSON，不发布 SQLite 数据库、后台程序或本机文件。
- `.github/workflows/update-pages.yml` 是自动更新与发布流程。

GitHub 定时任务不是精确实时任务，高峰期可能延迟。第一次发布还需要在仓库设置中允许 GitHub Pages 使用 Actions 发布。

### 服务器方式：容器部署

适合需要私有仓库、更高刷新稳定性或后续接入登录权限的场景。

项目已包含容器部署文件：

```bash
docker compose up -d --build
```

公网部署默认行为：

- 后台每 60 分钟自动采集。
- 所有访问者页面每 60 秒自动同步。
- 访客只能读取和搜索，公网环境关闭手动刷新接口。
- SQLite 数据库保存在服务器的 `data` 目录，服务重启后不会丢失。
- 服务器需保持运行并允许访问巨潮资讯和东方财富公开接口。

正式对外使用时，可再配置自定义域名。无论选择哪种方式，发布动作都需要对应托管平台的账号与权限。

## 维护命令

首次历史回填：

```bash
python3 cninfo_updater.py --start 2021-01-01 --skip-pdf
python3 hkex_updater.py --start 2021-01-01 --skip-pdf
```

补齐上市公司画像：

```bash
python3 eastmoney_profile_enricher.py
```

导出静态快照：

```bash
python3 export_records_js.py
```

抽取规则升级后重跑指定时间段：

```bash
python3 cninfo_updater.py --start 2026-07-01 --refresh-existing
```

## 主要能力

- 首页默认展示近 7 天新增、上市公司数、子公司事项、高价值线索和待复核事项。
- 按市场、企业性质、承租关系、地区、证监会行业、业务类型、业务阶段、金额、期限、出租人、租赁物和担保方式筛选。
- 基于公告披露字段生成业务机会分、风险关注分和信息完整度，并按项目机会排序。
- 公告列表与右侧证据链联动，展示业务要素、原文、PDF、命中片段和同一项目时间线。
- 提供上市公司雷达、同业雷达、趋势与结构分析、来源健康度页面。
- 收藏、工作状态、负责人、跟进日期、备注和常用筛选保存在当前浏览器。
- 默认只下载近 90 天数据；选择历史区间时按年度分片加载，全部历史从 2021 年开始。
- 导出当前筛选结果为 CSV，并支持事件专属链接。

## 字段边界

地区和证监会行业来自公开公司资料；金额、期限、出租人、租赁物、担保方式和业务阶段由公告字段及证据片段规则化提取。企业性质属于可解释的规则判定字段，详情页显示判定依据；金额和风险评分不是审计或授信结论。缺少可靠依据的字段统一标记为“待补充”，不使用猜测值。

港交所历史回填使用官方标题检索，标题级记录标记为“待补充正文”；每小时增量范围内的新 PDF 会自动提取正文并升级为“已复核官方公告”。互联网和公众号没有通用、稳定且可合法无人值守调用的统一接口，因此当前不宣称已完成自动采集。
